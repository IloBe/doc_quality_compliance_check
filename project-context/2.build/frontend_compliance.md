# Frontend Page Requirements — Compliance

**Page label:** Compliance Mapping System  
**Route:** `/compliance`  
**Owner persona:** `@frontend-eng`

## Purpose

Define and communicate which standards and regulatory mappings are active for governance checks.

## Functional requirements

- Show pre-title label in dashboard-style typography.
- Show page title with info icon toggle for "Why this page matters".
- Display active standard cards and status indicators.
- Display compliance alerts and governance manual context panel.

## Data and state

- Current implementation may use mock/static data for demo.
- Future-ready for backend mapping source integration.

## UX properties

- Standards must present status clearly (Active/In Review).
- Alert items should be readable and easily scannable for review sessions.

## Acceptance criteria

- Governance stakeholders can quickly identify active mappings.
- Page explains why mappings matter for pass/fail interpretation.
